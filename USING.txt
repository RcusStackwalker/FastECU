FastECU build, installation, and usage notes

Build requirements

FastECU uses Bazel 9.1.1, C++20, and Qt 6.8.3. Install Bazelisk (or the
version pinned in .bazelversion), Qt host tools with Qt Charts, Serial Port,
Remote Objects, and WebSockets, a platform C++ compiler, and OpenSSL.

Build the application from the repository root:

    bazel build --config=release //:fastecu

Run the maintained tests:

    bazel test --config=release //tests/... //:bazel_openssl_wiring

The exact Windows, macOS, and Linux dependency setup is kept in
.github/workflows/pr.yml. Release packages for Windows and macOS are assembled
from Bazel outputs by scripts/package-windows.ps1 and
scripts/package-macos.sh.

Installation

Download a package for a tagged release from:

    https://github.com/RcusStackwalker/FastECU/releases

Extract the archive to a convenient folder. Hardware communication and flash
operations remain adapter-, ECU-, and protocol-specific; review the applicable
bench checklist before using an experimental or unqualified path on a vehicle.

Basic use

Select the serial or J2534 adapter from the toolbar, then select the vehicle and
protocol before starting ECU operations. The Play action starts or stops
logging, and the Record action controls writing log values to a file.

The read, test-write, and write actions operate on the selected vehicle and
protocol. Always save and verify a known-good ROM before writing. Bench-qualify
new or experimental flash paths and have a recovery method available.

The Tune menu provides calibration-map editing commands. Copy and paste work
inside map views. The logging-view settings allow the displayed parameters and
gauges to be changed.

Known UI limitations

- Undo and redo commands are present but not implemented.
- Graph logging views are not implemented.
- Custom seed/access-method configuration is not implemented.

Safety

FastECU is development software for hardware that can be rendered unusable by
an interrupted or incorrect write. All liability lies with the user. The
project authors are not responsible for damage, legal violations, or bricked
ECUs resulting from use of the software.
